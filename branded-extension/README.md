# Branded Extension

A working browser extension (Chrome/Edge, Manifest V3) for
privacy-preserving payment verification on the **Peer** protocol. It is
**brand-neutral by design**: all identity lives in one config file, so you can
fork this and turn it into your own branded extension without touching the
engine.

Out of the box it ships as a fictional brand, **"Acme Verify"**, a placeholder
you replace with your own.

## What it does

The extension injects `window.peer` into your web app, and when the app calls
`peer.authenticate(...)` it:

1. opens the relevant payment-platform tab,
2. guides the user to the right transaction,
3. captures the payment session material from the page,
4. encrypts it in an offscreen document and submits it to the attestation
   service,
5. returns the verification result to your app via `peer.onMetadataMessage(...)`.

Passwords, cookies, and screenshots stay on the device. The extension returns
the attested result to your app. See [`docs/concept.md`](docs/concept.md) for
the architecture.

## Rebrand

**Prerequisites:** Node 18+ and npm. A Chromium browser for loading the result.

```bash
# 1. Install
npm install

# 2. Make it yours: edit one file
#    Open brand.config.json and set name, vendorId, theme colors,
#    hostDomains (the payment platforms you support) and appOrigins
#    (your web app's domains). Drop your own PNGs into icons/.
$EDITOR brand.config.json

# 3. Apply the brand across the manifest, theme, vendor marker and icons
npm run rebrand

# 4. Build
npm run build

# 5. Load it
#    Chrome → chrome://extensions → enable "Developer mode"
#    → "Load unpacked" → select the build/ folder
```

Re-run `npm run rebrand && npm run build` whenever you change
`brand.config.json`. `rebrand` is idempotent — running it without changes is a
no-op.

### What `brand.config.json` controls

| Field | Used for |
| --- | --- |
| `name`, `shortName`, `description` | Manifest + store identity, popup title, overlay label |
| `vendorId` | The `data-peer-vendor` page attribute (how a host app attributes which extension served a session) |
| `theme` | Popup + in-page overlay colors |
| `iconDir` | Folder `rebrand` reads icons from (default `icons/`) |
| `hostDomains` | Payment-platform domains the capture overlay may run on → narrows `host_permissions` |
| `appOrigins` | Your web app origins that may use `window.peer` (auto-approved, content script injected here) |
| `apiBaseUrl`, `attestationServiceUrl` | The Peer services the extension talks to (default to the public protocol endpoints) |

Read [`docs/host-permissions.md`](docs/host-permissions.md) before shipping.
narrow permissions are what get an extension through store review.

## Scripts

| Command | Description |
| --- | --- |
| `npm run rebrand` | Apply `brand.config.json` to the manifest, theme, vendor marker, and icons |
| `npm run build` | Production build into `build/` (load this unpacked) |
| `npm run dev` | Watch build for local development |
| `npm run typecheck` | `tsc --noEmit` |
| `npm run test` | Run the unit tests |
| `npm run release` | Build and zip `build/` into `release/<name>-<version>.zip` for the Web Store |

## Project layout

```
branded-extension/
  brand.config.json          ← the only file you normally edit
  scripts/rebrand.mjs         ← applies brand.config across the project (idempotent)
  scripts/zip-release.mjs     ← packages build/ for the Chrome Web Store
  icons/                      ← your 16/32/48/128 PNGs
  src/
    config/                   ← brand.ts (stable) + brand.generated.ts (rebrand output)
    manifest.json             ← generated by rebrand
    entries/Background/        ← service worker: capture orchestration, auth overlay
    entries/Content/           ← window.peer injection, connection approval, click guide
    entries/Offscreen/         ← capture encryption / credential bundling (isolated document)
    entries/Popup/             ← toolbar popup
    utils/                     ← capture engine, metadata engine, message types
  docs/                        ← concept, permissions, security invariants, integration
  templates/                   ← brand-free store listing / privacy / permission copy
```
